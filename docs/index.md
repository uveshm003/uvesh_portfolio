## Project Overview

`uvesh_portfolio` is the personal portfolio website of Muhammad Uvesh
Menpurwala, built with Flutter Web. It is a client-only single-page app with
seven routed sections under a persistent sidebar shell, light/dark themes, and a
distinctive "Instrument" design system. All content is compile-time constant
Dart in `lib/data/portfolio_data.dart`; there is no backend.

## Quick Reference

- **Tech Stack:** Flutter (Web), Dart ^3.11.0, go_router, google_fonts, url_launcher
- **Entry Point:** `lib/main.dart` → `MaterialApp.router` → `buildRouter` (`lib/router/app_router.dart`)
- **Architecture Pattern:** `ShellRoute` (fixed sidebar) + per-page `PageScaffold`; token-driven theme
- **Database:** none
- **Deployment:** Netlify (static `build/web`, SPA fallback)

## Getting Started

### Prerequisites

Flutter SDK (stable) with Dart ^3.11.0; web enabled (`flutter config --enable-web`); a Chromium browser.

### Setup

```bash
flutter pub get
```

### Run Locally

```bash
flutter run -d chrome
```

### Run Tests

```bash
flutter test
```

## For AI-Assisted Development

This documentation was generated to help AI agents understand and extend this
codebase.

### When planning new features

- **UI / content changes:** Reference `architecture.md`, `component-inventory.md`,
  and edit content in `lib/data/portfolio_data.dart`.
- **New section/route:** add a `NavItem` (`portfolio_data.dart`), a `GoRoute`
  (`router/app_router.dart`), and a page (`pages/`) — see the Development Guide.
- **Look & feel:** change tokens in `lib/theme/` (`app_colors`, `app_typography`,
  `app_spacing`); everything reads from these.
- **Deployment changes:** Reference `deployment-guide.md` and `netlify.toml`.

### Key invariants

- Content lives only in `portfolio_data.dart`.
- One `FadeIn` entrance per page (in `PageScaffold`); don't nest in sections.
- The Reading section's route is `/books`.
- `lib/widgets/top_nav.dart` is dead code (superseded by `sidebar.dart`).
